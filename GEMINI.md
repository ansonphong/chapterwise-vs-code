# Gemini CLI — Local Wrapper for chapterwise-vs-code

This repo uses a local `CLAUDE.md` for build, run, and test instructions, and a local `.claude/` for context. Gemini CLI is configured as a wrapper for this system.

## Canonical Instructions

**Gemini MUST follow the instructions in this repo's:**
1. **`CLAUDE.md`** — The master project guide, brand voice, and waterfall process for this repo.
2. **`.claude/`** — Local context, settings, and references.

## Core Mandates

- **Strategy:** Always defer to the parent repo's `GEMINI.md` and `CLAUDE.md` for project-level strategy, plans, and status tracking.
- **Workflow:** Strictly follow the **6-stage Development Waterfall** described in the root `CLAUDE.md`.
- **Maintenance:** After execution, follow the "Post-Plan Completion Workflow" in the root `CLAUDE.md`, updating both local and parent context files.

## Session Quick-Start (Local)

1. **Read local `CLAUDE.md`** for build/run/test instructions.
2. **Read local `.claude/context/`** for module-level architecture.
3. **Check the parent repo** for the latest plans and status.
